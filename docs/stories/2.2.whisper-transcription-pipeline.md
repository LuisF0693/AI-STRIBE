## Story
**As a** produto,
**I want** processar o áudio de cada consulta com Whisper fine-tuned em português médico,
**so that** a transcrição gerada tenha precisão ≥92% com terminologia clínica correta e seja entregue ao médico em menos de 60 segundos após o encerramento da consulta.

## Acceptance Criteria
1. Worker assíncrono consome fila BullMQ `transcription-queue` após upload de áudio concluído (Story 2.1)
2. Integração com OpenAI Whisper API (`whisper-1` ou modelo fine-tuned PT-BR) configurada via variável de ambiente `OPENAI_API_KEY`
3. Áudio dividido em chunks de máximo 25MB antes do envio à API (limite OpenAI) — chunking transparente para o resultado final
4. Transcrição retornada com timestamps por segmento (word-level timing para destacar trechos na UI)
5. Resultado da transcrição salvo na tabela `transcricoes` com: `consulta_id`, `texto_completo`, `segmentos_json`, `duracao_ms`, `status`
6. Status da transcrição atualizado em tempo real via Supabase Realtime para o app mobile acompanhar progresso
7. Tempo total de processamento (upload concluído → transcrição disponível) ≤ 60 segundos para consultas de até 30 minutos
8. Em caso de erro na API OpenAI, retry automático com backoff exponencial (max 3 tentativas) antes de marcar como `failed`
9. Custo por transcrição logado em `transcricoes.custo_usd` para controle de billing
10. Transcrição disponível mesmo offline — texto armazenado localmente no device após sync

## Dev Notes

### Contexto Técnico
- Dependência direta: Story 2.1 deve estar concluída (tabela `consultas` + arquivo no Storage)
- BullMQ + Redis: Redis deve estar provisionado — usar Upstash Redis (plano free, região BR disponível)
- OpenAI Whisper: modelo `whisper-1` suporta PT-BR nativamente com boa precisão. Fine-tuned é roadmap futuro — iniciar com `whisper-1` + prompt de contexto médico para melhorar precisão
- Prompt de contexto sugerido: `"Transcrição de consulta médica em português brasileiro. Terminologia: CID-10, TUSS, prontuário, anamnese, hipótese diagnóstica."`

### Chunking de Áudio
- Limite da OpenAI API: 25MB por request
- Consulta de 30min em `.m4a` 16kHz mono ≈ 30MB → precisa dividir em 2 chunks
- Usar `ffmpeg-static` para split sem perda de qualidade
- Concatenar transcrições dos chunks preservando timestamps contínuos

### Referências de Schema
```sql
CREATE TABLE transcricoes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  consulta_id UUID REFERENCES consultas(id) ON DELETE CASCADE,
  texto_completo TEXT,
  segmentos_json JSONB,  -- [{start, end, text, words: [{word, start, end}]}]
  duracao_ms INTEGER,
  status TEXT CHECK (status IN ('pending', 'processing', 'completed', 'failed')),
  custo_usd DECIMAL(10,6),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
