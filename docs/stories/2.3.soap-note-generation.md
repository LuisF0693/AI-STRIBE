## Story
**As a** médico,
**I want** que o sistema gere automaticamente uma nota clínica estruturada em formato SOAP a partir da transcrição da consulta,
**so that** eu tenha um rascunho completo pronto para revisar em vez de escrever do zero, economizando 15-20 minutos por consulta.

## Acceptance Criteria
1. Após transcrição concluída (Story 2.2), worker dispara geração de nota SOAP via GPT-4o automaticamente
2. Nota gerada contém os 4 campos SOAP: **S** (Subjetivo), **O** (Objetivo), **A** (Avaliação/Hipótese Diagnóstica), **P** (Plano/Conduta)
3. Campo **A** inclui sugestão de até 3 CIDs-10 prováveis com código e descrição (ex: "J00 — Rinofaringite aguda")
4. Nota gerada em português brasileiro formal, com terminologia médica padronizada (CFM)
5. Campos da nota são estruturados como JSON para permitir edição granular na UI (Story 3.x)
6. Nota salva na tabela `notas` com: `consulta_id`, `transcricao_id`, `soap_json`, `cids_sugeridos`, `status`, `versao`
7. Tempo de geração da nota ≤ 30 segundos após transcrição disponível
8. Em caso de transcrição de baixa qualidade (< 50 palavras), nota é gerada com aviso de `baixa_confianca: true`
9. Supabase Realtime notifica o mobile quando nota estiver pronta
10. Versioning da nota: cada edição cria nova versão em `notas_versoes` — nota original nunca é sobrescrita

## Dev Notes

### System Prompt (rascunho inicial)
```
Você é um assistente médico especializado em documentação clínica brasileira.
Com base na transcrição de consulta fornecida, gere uma nota clínica estruturada no formato SOAP.

REGRAS:
- Use terminologia médica em português brasileiro (padrão CFM)
- Campo S (Subjetivo): queixas, histórico relatado pelo paciente
- Campo O (Objetivo): dados observados, exame físico mencionado, sinais vitais
- Campo A (Avaliação): hipótese diagnóstica + sugerir até 3 CID-10 prováveis com código
- Campo P (Plano): conduta, prescrições mencionadas, solicitações de exame, retorno

FORMATO DE SAÍDA (JSON):
{
  "subjetivo": "...",
  "objetivo": "...",
  "avaliacao": "...",
  "plano": "...",
  "cids_sugeridos": [{"codigo": "J00", "descricao": "Rinofaringite aguda", "confianca": 0.92}],
  "baixa_confianca": false
}
```

### Schema de Referência
```sql
CREATE TABLE notas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  consulta_id UUID REFERENCES consultas(id),
  transcricao_id UUID REFERENCES transcricoes(id),
  soap_json JSONB NOT NULL,
  cids_sugeridos JSONB,
  status TEXT CHECK (status IN ('draft', 'reviewed', 'approved')),
  baixa_confianca BOOLEAN DEFAULT false,
  versao INTEGER DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE notas_versoes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  nota_id UUID REFERENCES notas(id),
  soap_json JSONB NOT NULL,
  versao INTEGER,
  editado_por UUID REFERENCES medicos(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Custo Estimado GPT-4o
- Transcrição média de 30min ≈ 3.000-5.000 tokens de input
- Output da nota ≈ 500-800 tokens
- Custo GPT-4o: ~US$0.01 por nota (input $2.50/1M + output $10/1M tokens)
- Meta: ≤ R$ 0,50/nota para viabilidade com plano de R$ 200/mês (400 consultas/mês = custo AI ≤ R$ 200)
