## Story
**As a** médico,
**I want** exportar a nota clínica aprovada como PDF com assinatura digital ICP-Brasil,
**so that** eu tenha um documento legalmente válido e compatível com CFM para arquivar no prontuário do paciente ou compartilhar com outros profissionais.

## Acceptance Criteria
1. Endpoint `POST /api/v1/notas/:id/exportar-pdf` dispara geração assíncrona via fila BullMQ `pdf-generation` — retorna imediatamente com `{ job_id, status: 'pending' }` (202 Accepted)
2. Worker `PdfGenerationWorker` consome a fila e gera PDF com: nome e CRM do médico, data e duração da consulta, nota SOAP formatada (S, O, A, P), CIDs aprovados, rodapé legal e badge LGPD
3. Assinatura digital ICP-Brasil integrada via Certisign API — credenciais (`CERTISIGN_API_KEY`, `CERTISIGN_CERT_ID`) gerenciadas exclusivamente via variáveis de ambiente Railway
4. PDF assinado é armazenado no Supabase Storage (bucket `notas-pdf`, sa-east-1, privado) com path `{medico_id}/{consulta_id}/nota-{versao}.pdf`
5. Após upload, o campo `notas.pdf_status` é atualizado para `'ready'` e `notas.pdf_url` (path interno) é salvo
6. Endpoint `GET /api/v1/notas/:id/pdf-url` retorna signed URL com TTL de 24 horas — nunca URL permanente
7. Endpoint `GET /api/v1/consultas/:id/pdf-status` retorna estado atual: `{ status: 'none'|'pending'|'processing'|'ready'|'failed', pdf_url?: string }` (polling pelo mobile)
8. Circuit breaker: se Certisign API falhar após 3 tentativas, PDF é gerado sem assinatura digital e `notas.pdf_status` é atualizado para `'ready_unsigned'` — médico notificado via push notification
9. Retry automático com backoff exponencial: 3 tentativas (30s, 2min, 10min) antes de acionar circuit breaker
10. Credenciais da Certisign NUNCA aparecem em logs, stack traces, ou retornos de API — scan automático no Pre-Commit

## Escopo

**IN (esta story):**
- Geração assíncrona de PDF via BullMQ `pdf-generation`
- Integração com Certisign API (ICP-Brasil)
- Circuit breaker com fallback para PDF sem assinatura
- Upload para Supabase Storage (bucket privado, sa-east-1)
- Signed URL com TTL de 24h
- Tela mobile de status de geração e download do PDF
- Migration 008 (pdf_url, pdf_signed)

- Aprovação da nota → Story 3.2 (pré-requisito)

- Assinatura digital do paciente → pós-MVP
- Múltiplos formatos de exportação (Word, HL7) → pós-MVP
- Template de PDF por especialidade → Epic 4
